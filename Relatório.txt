Relatório Técnico Completo — GestaoPC Storage Manager
Versão: 0.3.0 · Python: 3.11+ · Plataforma: Windows 11
Atualizado em: 2026-05-08

====================================================================
1. VISÃO GERAL
====================================================================
O GestaoPC Storage Manager é um aplicativo desktop avançado para gerenciamento de armazenamento otimizado para Windows 11. Ele mapeia discos físicos e lógicos de forma automática, detecta duplicatas com verificação criptográfica e sugere ações de realocação usando um Motor de Regras.

O aplicativo evoluiu de um sistema estritamente transitório para um ecossistema com base de dados persistente em SQLite, ganhou a capacidade de ser manipulado por Inteligências Artificiais externas via Servidor MCP (Model Context Protocol) e passou a contar com um Assistente IA Local integrado via Ollama.

====================================================================
2. FUNCIONALIDADES PRINCIPAIS
====================================================================
- Mapeamento Inteligente: Detecção de discos, partições, tipo de mídia (NVMe, SSD, HDD) e limites de uso.
- Varredura de Alto Desempenho: Identificação dos "Top 50 Maiores Arquivos" e "Top 20 Pastas" usando QThreads sem travar a interface.
- Algoritmo Anti-Duplicatas (3 Etapas): Compara por tamanho exato, hash de amostras das pontas (2MB) e finalmente hash SHA-256 completo.
- Motor de Regras Inteligentes: Identifica padrões (ex: arquivo de mídia de 50GB em um NVMe veloz que deveria estar em um disco SATA lento de backup).
- Histórico Persistente: Operações como Mover/Deletar são registradas em um banco de dados SQLite local, garantindo um "Undo" seguro.
- Servidor MCP Completo: Expõe 13 tools (7 leitura + request_confirmation + 5 executivas) e 3 resources JSON para clientes LLM externos via Model Context Protocol. Ações executivas auditadas com ai_source='ai:mcp'.
- Assistente IA Local (Ollama): Aba de chat integrada à GUI que conecta diretamente a modelos LLM locais (llama3, qwen2.5, etc.) rodando via Ollama, com contexto dinâmico do estado dos discos injetado automaticamente.

4. TECNOLOGIAS E DEPENDÊNCIAS
====================================================================
- Linguagem Base: Python 3.11+
- Banco de Dados: SQLite3 integrado nativamente.
- Framework GUI: PySide6 (QWidgets, QThread, QPainter) — LGPL v3, compatível com licença MIT do projeto.
- File System IO: psutil (monitoramento físico do HD/SSD) e bibliotecas Nativas.
- Antivírus / Sistema: Tratamento rigoroso de permissões e envios via `send2trash`.
- Protocolo IA (Externo): `mcp` (Model Context Protocol) provido pela Anthropic via SDK.
- IA Local: `ollama` via HTTP REST (porta 11434) — compatível com llama3, qwen2.5 e outros.
- Qualidade: pytest + pytest-cov (Testes), ruff (Linting), mypy (Type checking).

====================================================================
5. DETALHAMENTO DE PERSISTÊNCIA (SQLite)
====================================================================
Através do módulo `StorageManagerDB` e o seu banco em `%LOCALAPPDATA%/GestaoPC/storage_manager.db`, o app armazena as seguintes entidades:
- app_settings: Configurações dinâmicas persistidas.
- disk_roles: Rótulo lógico dos discos físicos.
- scan_sessions: Resumo contendo duração de cada varredura de discos.
- operation_history: Histórico de deleção/remoção persistido (Data, Caminho, Sucesso/Erro, Backup Lixeira).
- file_index: Um índice/cache com arquivos de interesse com a sua respectiva categoria, hashes e timestamps.
- suggestions: Recomendação do motor guardadas em cache temporário após cada varredura.

====================================================================
6. SEGURANÇA E PROTEÇÃO
====================================================================
- Resiliência contra Antivírus: Pastas bloqueadas temporariamente por Kaspersky e outros não "quebram" o sistema de varredura.
- Exclusão do Core do SO: Diretórios vitais do Windows (`C:\Windows`, `C:\Program Files`, `ProgramData`) são hardcoded para serem ignorados.
- Undo Reversível: Arquivos enviados para a Lixeira do Windows podem ser resgatados facilmente; Movimentações registradas no SQLite podem ser desfeitas via interface do app.

====================================================================
7. INSTRUÇÕES E COMANDOS DE USO
====================================================================
O software contém ferramentas em linha de comando interativas para testes, uso pessoal e interface gráfica. Para a melhor experiência, ative seu ambiente virtual `venv`.

A) Instalação das Dependências:
> pip install -e .
Ou se preferir métodos isolados:
> pip install -r requirements.txt

B) Inicializar o Aplicativo Gráfico (Interface):
> python run_gui.py
Para pular o clique do botão manual e começar a mapear o PC imediatamente:
> python run_gui.py --auto-scan

C) Inicializar o Servidor MCP (Para Clientes LLM como Claude Desktop):
> python -m scripts.mcp_server
Nota: Adicione ao arquivo de configuração de seu cliente MCP as opções `command: python`,
`args: ["-m", "scripts.mcp_server"]` tendo o cwd focado nesta pasta raiz.

D) Usar o Assistente IA Local (Ollama):
Pré-requisito: instale o Ollama (https://ollama.com) e baixe um modelo:
> ollama run llama3
Após iniciar o app normalmente (item B), acesse a aba "Assistente IA" na GUI.

E) Execução da Suíte de Testes (TDD):
Para testar a segurança contra regressão:
> python -m pytest tests/ -v
Para relatório de cobertura:
> python -m pytest tests/ --cov=src --cov=scripts --cov-report=term-missing -q
